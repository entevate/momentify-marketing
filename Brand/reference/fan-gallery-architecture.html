System Architecture
Authentication
@clerk/nextjs ClerkProvider + middleware
middleware.ts Protects /fan-gallery/admin/*
Organizations Multi-tenant workspace isolation
Webhook Syncs orgs to Postgres
Database
@vercel/postgres Managed PostgreSQL
drizzle-orm Type-safe schema + queries
drizzle-kit Migrations + generation
JSONB Gallery config stored as document
File Storage
@vercel/blob Photo uploads (existing pattern)
sessions.photo_url Blob URL in Postgres
QR SVG/PNG Generated server-side
Admin Frontend
Level 1 Events list (TabBar, EventGrid)
Level 2 Event detail + sessions table + moments drawer
Level 3 Moment editor (existing admin page)
Reuse Dashboard components (TabBar, DrawerPanel, etc.)
QR Code Service
qrcode Server-side SVG/PNG generation
nanoid 8-char short codes for tracking
Redirect API /api/fan-gallery/qr/[code]
Dynamic Update target URL without reprint
Public Gallery
/fan-gallery/[slug] No auth required
4-screen wizard Gallery, Capture, Contact, Confirm
Real sessions Photo + data to Postgres/Blob
My Moments Session-based, DB-backed
Notifications (Email + SMS)
@sendgrid/mail Transactional email via SendGrid
twilio SMS delivery via Twilio
Pre-submission Gallery link sent after contact form (Screen 2)
Post-submission Photo ready + download link + sponsor CTA
Triggers Contact form submit → instant email + SMS
Tracking email_sent, sms_sent, notification_sent_at per session
